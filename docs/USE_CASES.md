# Use Cases

Common use cases and examples for Little Boys e-commerce platform.

## Customer Use Cases

### 1. Browse Products by Category
**Scenario**: Customer wants to find toys for their toddler

**Steps**:
1. Visit homepage
2. Click on "Toys & Games" category
3. Filter by age range (2-4 years)
4. Sort by rating or price
5. View product details
6. Add to cart

### 2. Search for Specific Product
**Scenario**: Customer looking for "wooden blocks"

**Steps**:
1. Click search icon in header
2. Enter "wooden blocks"
3. View search results
4. Filter by category or age
5. Select product and purchase

### 3. Read Product Reviews
**Scenario**: Customer wants to check product quality

**Steps**:
1. Navigate to product detail page
2. Scroll to reviews section
3. Read customer reviews
4. Check ratings and verified purchases
5. Write own review (optional)

### 4. Manage Shopping Cart
**Scenario**: Customer adding multiple items

**Steps**:
1. Browse products
2. Add items to cart
3. Open cart sidebar
4. Update quantities
5. Remove unwanted items
6. Proceed to checkout

### 5. Save Items for Later
**Scenario**: Customer not ready to purchase

**Steps**:
1. Browse products
2. Click "Add to Wishlist"
3. Continue shopping
4. Access wishlist from header
5. Move items to cart when ready

## Content Use Cases

### 6. Read Blog Articles
**Scenario**: Parent seeking parenting tips

**Steps**:
1. Navigate to Blog
2. Browse articles or search
3. Filter by category
4. Read article
5. Share on social media
6. Read related articles

### 7. Learn About Company
**Scenario**: Customer wants to know company values

**Steps**:
1. Navigate to About page
2. Read company story
3. View team members
4. Check sustainability practices
5. Read customer testimonials

## Developer Use Cases

### 8. Add New Feature
```bash
# Create feature branch
git checkout -b feature/new-feature

# Develop and test
npm run dev

# Commit and push
git commit -m "feat: add new feature"
git push origin feature/new-feature
```

### 9. Fix Bug
```bash
# Create bug fix branch
git checkout -b fix/bug-description

# Fix and test
npm run build

# Commit and push
git commit -m "fix: resolve bug"
```

### 10. Deploy to Production
```bash
# Build production bundle
npm run build

# Test production build
npm run preview

# Deploy to hosting service
```

These use cases cover common scenarios for customers and developers.
